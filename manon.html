From Sandbox Experience to Management Game

My work focused on transforming the shared interactive foundation into a true management game experience. I reintegrated surface detection features to improve spatial stability and implemented a "Rush" gameplay loop including thermal quality constraints.

My Individual Contributions

Focus on technical architecture and game design mechanics.

1. Native Plane Detection

Instead of relying exclusively on Hit-Test (which requires continuous targeting), I implemented native plane detection. The headset memorizes the room geometry, allowing customers and objects to remain firmly anchored without spatial drift. 

// WebXR initialization with plane detection
this.el.sceneEl.setAttribute('webxr', {
  requiredFeatures: ['planes', 'anchors', 'local-floor'],
  optionalFeatures: ['hit-test']
});

this.el.addEventListener('planesdetected', (ev) => {
  const planes = ev.detail.planes;
  // Geometry memorization
});

2. Temperature Gauge System

Each coffee has a temperature variable that decreases over time. A world-space UI gauge visually indicates its state: if the temperature drops below 30%, the customer refuses the order. 

// Heat loss management
tick: function(time, delta) {
  this.temperature -= this.coolingRate * (delta / 1000);
  
  // Update world-space gauge
  this.gaugeEl.setAttribute('width', this.temperature / 100);

  if (this.temperature < 30) this.isCold = true;
}

3. Queue Management System

Development of a rush mechanic where NPC customers dynamically line up behind one another, creating immediate visual pressure and reinforcing the management gameplay loop.
